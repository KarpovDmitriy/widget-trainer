# **Описание работы**

## Widget Trainer

Веб-приложение для обучения и тренировок, построенное на **React 19**, **Vite** и **TypeScript**. Проект представляет собой масштабируемую платформу с интерактивными виджетами, системой авторизации и продвинутой локализацией.

## Развернутый обзор архитектуры

Проект представляет собой масштабируемое **Single Page Application (SPA)**, построенное по принципам модульности и строгой типизации.

### Ключевые архитектурные решения:

1.  **State Management:** Использование **Zustand** позволило создать атомарные и производительные хранилища (Auth, Profile, Theme, Notifications).
2.  **Type Safety:** Проект полностью типизирован. Связка **Zod + TypeScript** гарантирует консистентность данных на всех этапах: от валидации схем до пропсов UI-компонентов.
3.  **Internationalization (i18n):** Система локализации пронизывает все уровни. Динамические справочники стран и таймзон (ISO) генерируются через **Intl API**, что обеспечивает мультиязычность без раздувания бандла.
4.  **Adaptive Design & UX:**
    - Интерфейс полностью адаптивен **вплоть до разрешения 320px**.
    - Реализовано бургер-меню с программной блокировкой скролла.
    - Используются **CSS Modules** для полной изоляции стилей.
5.  **DX (Developer Experience):** Настроен конвейер проверок кода через **Husky и Lint-staged** (ESLint, Stylelint, Prettier), а также соблюдение стандартов **Conventional Commits** через Commitlint.

# **2 личных Feature Component — компоненты, которые разработала лично**

## Key Contributions (Личная разработка и проектирование)

В рамках проекта я выделила два модуля, которые были полностью спроектированы и реализованы мной «с чистого листа».

### 1. Система динамических уведомлений (Toast Notification System)

Это независимый сервис внутри приложения для информирования пользователя, работающий по принципу «вызвал и забыл» из любой точки кода.

- **Архитектурное решение (State Management):**
  - Создан кастомный стор на **Zustand**, который управляет массивом активных уведомлений.
  - Реализован метод `addToast`, который автоматически генерирует уникальный `ID` на основе `Date.now()`, что исключает коллизии при быстром появлении нескольких сообщений.
  - Внедрена логика **автоматического удаления**: каждое уведомление живет заданное время (по умолчанию 4000мс), после чего стор сам инициирует процесс закрытия.

- **UX и анимации (Life-cycle):**
  - **Двухэтапное удаление:** Я реализовала метод `dismissToast`, который сначала устанавливает флаг `isExiting: true`. Это позволяет CSS-анимации плавно скрыть элемент (300мс) до того, как он будет окончательно вырезан из DOM методом `removeToast`. Это решает проблему «дерганого» исчезновения интерфейса.
  - **React Portals:** Контейнер уведомлений рендерится в `document.body`. Это гарантирует, что уведомления всегда будут поверх модальных окон, хедеров и других элементов с высоким `z-index`.

- **Интернационализация:**
  - Компонент `ToastItem` проверяет входящее сообщение через `i18n.exists()`. Если передан ключ — выводится перевод, если сырая строка (например, ошибка от сервера) — выводится текст как есть.

- **Сложность:** Синхронизация асинхронных таймеров в JS с длительностью CSS-переходов, чтобы избежать утечек памяти и артефактов верстки.

### 2. Модуль профиля пользователя (Comprehensive Profile Module)

Сложная многофункциональная страница, объединяющая просмотр данных, их редактирование и синхронизацию с глобальным состоянием.

- **Управление данными и состоянием:**
  - **Lazy Loading:** Реализована логика первичной подгрузки данных при переходе на страницу. С помощью `useRef(isFirstLoad)` я предотвратила повторные тяжелые запросы к API при переключении между вкладками «Обзор» и «Настройки».
  - **Интеграция со стором:** Профиль напрямую связан с `useProfileStore`. После успешного сохранения данных состояние обновляется глобально, что мгновенно отражается в хедере (обновление инициалов аватара).

- **Формы и Валидация (Advanced RHF + Zod):**
  - **Умная блокировка:** Кнопка «Сохранить» активируется только при соблюдении двух условий: форма валидна (`isValid`) и данные были изменены (`isDirty`). Это исключает пустые запросы к серверу.
  - **Отмена изменений:** Реализована функция `handleCancel`, которая сбрасывает состояние формы к исходным значениям из стора одним кликом.

- **Трансформация ISO-данных (Data Normalization):**
  - В режиме просмотра я внедрила кастомную логику (через хук `useSelectOptions`), которая превращает технические коды (например, `ru`, `UTC+3`, `US`) в полные локализованные названия. Это делает профиль читаемым и профессиональным.

- **Адаптивность (Mobile First):**
  - Верстка страницы спроектирована так, что на разрешении **320px** двухколоночные строки (`firstName` / `lastName`) перестраиваются в одну колонку, сохраняя читабельность и удобство клика.

- **Сложность:** Проектирование единого интерфейса `UserData`, который используется одновременно в схемах валидации Zod, дефолтных значениях формы и при отображении данных в режиме Read-only.
